## 系统整体架构

```
Controller (10.130.131.239:8000)
  │
  ├── POST /int                        ← 接收各网元 INT 数据，检测攻击
  ├── POST /game-theory/apply-optimal  ← 博弈论选取最优策略并下发
  ├── POST /mimetic/start              ← 手动 SSH 远程启停拟态方法
  ├── POST /mimetic/stop
  ├── GET  /mimetic/status
  ├── GET  /defense/dashboard          ← 总览 API（前端核心数据源）
  │
  ├── SSH → s1 (192.168.1.2)  → 拟态入侵检测
  ├── SSH → s4 (192.168.4.2)  → 拟态自适应防御
  └── SSH → s7 (192.168.7.2)  → 拟态蜜罐
```

| 类别 | 部署位置 | 启动方式 |
|------|---------|---------|
| 拟态入侵检测 | **s1 网元** | s1 本地 `mimic_start.py start` 或 Controller SSH 远程启动 |
| 拟态自适应防御 | **s4 网元** | s4 本地 `mimic_start.py start` 或 Controller SSH 远程启动 |
| 拟态蜜罐 | **s7 网元** | s7 本地 `mimic_start.py start` 或 Controller SSH 远程启动 |
| 博弈论选取 | **Controller** | Controller 本地 `controller_server_v2.py` |

## 各方法启动文件路径与启停命令速查

### 1. 拟态入侵检测（部署在 s1）

| 项目 | 内容 |
|------|------|
| **网元** | s1（入口交换机） |
| **SSH IP** | 192.168.1.2（实验网）/ 10.130.131.254（管理网） |
| **项目路径** | `/home/p4/njupt_mimic_detection/code/` |
| **启停脚本** | `mimic_start.py`（位于上述路径） |

```bash

cd /home/p4/njupt_mimic_detection/code

# 启动检测系统（首次需加载 4 个模型，约 10-20 秒）
```

## 四个页面的数据来源与展示内容

### 页面 1：拟态入侵检测

**数据来源 API：**

| API | 方法 | 说明 |
|-----|------|------|
| `GET /defense/dashboard` | Controller | 查看 `mimetic_status.mimetic_intrusion_detection` 获知运行状态 |
| `GET /mimetic/status` | Controller | 查看拟态入侵检测的 SSH 状态检查结果 |
| **s1 本地日志** | s1 SSH 读取 | `~/njupt_mimic_detection/mimic_detection.log` |
| **s1 恶意包目录** | s1 SSH 读取 | `~/njupt_mimic_detection/capture_traffic/malicious/` |
| **攻击测试** | host1 SSH 执行 | `send_pcap.py`（详见下方 [攻击测试章节](#1-测试拟态入侵检测host1--s1)） |

**前端需展示：**

1. **运行状态** — 检测系统是否运行中（绿/红灯）
2. **近期恶意流量列表** — 读取 s1 上的日志，筛选含 `⚠ MALICIOUS` 的行：
   ```bash
   # 日志示例行
   tail -100 ~/njupt_mimic_detection/mimic_detection.log | grep "MALICIOUS"
   ```
   日志中每条 `⚠ MALICIOUS` 表示检测到一个恶意包。展示字段建议：
   - 时间戳
   - 攻击类型（从文件名或日志中提取）
   - 来源场景（GEO_P4 / IPv4_P4 / IPv6_P4 / MPLS_P4 / SCION_P4）
   - 裁决结果
3. **恶意包文件列表** — 列出 `capture_traffic/malicious/` 下的文件：
   ```bash
   ls -lt ~/njupt_mimic_detection/capture_traffic/malicious/ | head -20
   ```
   文件名格式：`malicious_<场景>_<攻击类型>_<pcap名>_<时间戳>.pcap`
4. **检测统计** — 恶意包总数、各攻击类型分布
   ```bash
   grep -c "MALICIOUS" ~/njupt_mimic_detection/mimic_detection.log
   ```
5. **模型信息** — 当前上线的 4 个异构模型（CNN/GRU/DNN/MLP）及裁决参数
6. **启停按钮** — 调用 `POST /mimetic/start` / `POST /mimetic/stop`（`strategy_name: mimetic_intrusion_detection`）

### 页面 2：拟态防御

**数据来源 API：**

| API | 方法 | 说明 |
|-----|------|------|
| `GET /defense/dashboard` | Controller | 查看 `mimetic_status.mimetic_adaptive_defense` |
| `GET /mimetic/status` | Controller | 查看拟态防御远程状态 |
| **s4 本地命令** | s4 SSH 执行 | `python mimic_start.py status` |
| **攻击测试** | host1 SSH 执行 | `curl` + `attack.py`（详见下方 [攻击测试章节](#2-测试拟态防御host1--s4-内联代理-8080)） |

**前端需展示：**

1. **运行状态** — 平台是否运行中（绿/红灯）
2. **当前上线执行体列表** — 通过 `mimic_start.py status` 获取：
   - Voting 服务状态（`:5000/health`）
   - 内联代理状态（`:8080`）
   - 异构体容器列表（5 个 `hetero_*` 容器）及各容器信息
   ```bash
   # s4 上查看容器
   docker ps | grep hetero
   ```
3. **优化状态** — 展示 REMI-AIA 算法是否已完成优化：
   - 优化前的防御成功率（约 50-70%）
   - 优化后的防御成功率（约 80-100%）
   - REMI 优化迭代次数
4. **防御统计** — 总攻击数 / 阻断数 / 放行数 / 防御成功率
   ```bash
   # 查看内联代理日志
   tail -50 ~/njupt_micic_defense/REMI-AIA/logs/inline_proxy.log
   ```
   日志关键行：
   - `✓ 流量放行` — 正常流量通过
   - `🛡 攻击阻断` — 攻击被拦截（含路径、裁决置信度）
5. **裁决机制展示** — 3+1 异构体投票裁决原理图（N/2 攻破即判定攻击）
6. **启停按钮** — 调用 `POST /mimetic/start` / `POST /mimetic/stop`（`strategy_name: mimetic_adaptive_defense`）

### 页面 3：拟态蜜罐

**数据来源 API：**

| API | 方法 | 说明 |
|-----|------|------|
| `GET /defense/dashboard` | Controller | 查看 `mimetic_status.mimetic_honeypot` |
| `GET /mimetic/status` | Controller | 查看拟态蜜罐远程状态 |
| **s7 本地命令** | s7 SSH 执行 | `python3 mimic_start.py stats` |
| **s7 数据库** | s7 MySQL 查询 | `es_honeypot` 表 |
| **攻击测试** | host1 SSH 执行 | `attack_mimic_honeypot.py`（详见下方 [攻击测试章节](#3-测试拟态蜜罐host1--server1-数据面-ip)） |

**前端需展示：**

1. **运行状态** — 蜜罐服务是否运行中（绿/红灯）、监听端口 9200、Nginx 状态、tshark 抓包状态
2. **诱捕攻击列表** — 从 s7 数据库或 `mimic_start.py stats` 获取：
   ```bash
   # s7 上执行
   python3 mimic_start.py stats
   ```
   或直接查数据库：
   ```sql
   SELECT pk_id, attacker, type, date FROM es_honeypot ORDER BY pk_id DESC LIMIT 20;
   ```
   展示字段建议：
   - 攻击 ID
   - 攻击者 IP
   - 攻击类型（SQL注入 / Apache_CVE / Nmap扫描 / 目录扫描 / XSS 等）
   - 攻击时间
3. **诱捕率** — 计算公式：
   ```
   诱捕率 = 蜜罐检测到的攻击数 / 总发送攻击数 × 100%
   ```
   攻击统计中的类型分布与总攻击数：
   ```sql
   SELECT type, COUNT(*) as cnt FROM es_honeypot GROUP BY type ORDER BY cnt DESC;
   ```
4. **攻击类型分布图** — 饼图或柱状图展示各类攻击占比
5. **实时日志流** — 可选展示 tshark 转发日志：
   ```bash
   tail -f /tmp/tshark_feed.log
   ```
6. **启停按钮** — 调用 `POST /mimetic/start` / `POST /mimetic/stop`（`strategy_name: mimetic_honeypot`）

## 前端 API 调用汇总

所有 API 的基础 URL：`http://10.130.131.239:8000`

### 核心总览 API

| 端点 | 方法 | 用途 | 前端页面 |
|------|------|------|---------|
| `/health` | GET | Controller 健康检查 | 全部 |
| `/defense/dashboard` | GET | **总览**：攻击状态 + 博弈结果 + 拟态状态 | 博弈论（主要）、全部 |
| `/status` | GET | 各网元攻击状态 + 当前模态 | 博弈论 |
| `/mimetic/status` | GET | 3 种拟态方法运行状态 | 全部 |

### 博弈论 API

| 端点 | 方法 | 请求体 | 用途 |
|------|------|--------|------|
| `/game-theory/optimal-strategy` | POST | `{"switch_ids":[1,3,4,7]}` | 查询最优策略（不执行） |
| `/game-theory/apply-optimal` | POST | `{"switch_ids":[1,3,4,7]}` | 选取并应用最优策略 |
| `/game-theory/combination` | POST | `{}` | 传统+拟态组合推荐 |

### 拟态控制 API

| 端点 | 方法 | 请求体 | 用途 |
|------|------|--------|------|
| `/mimetic/start` | POST | `{"strategy_name":"mimetic_honeypot"}` | 启动指定拟态方法 |
| `/mimetic/stop` | POST | `{"strategy_name":"mimetic_honeypot"}` | 停止指定拟态方法 |
| `/mimetic/stop-all` | POST | `{}` | 一键停止全部拟态 |

`strategy_name` 可用值：
- `mimetic_intrusion_detection` — 拟态入侵检测 (s1)
- `mimetic_adaptive_defense` — 拟态自适应防御 (s4)
- `mimetic_honeypot` — 拟态蜜罐 (s7)

### 模拟攻击 API（用于测试）

| 端点 | 方法 | 请求体示例 | 用途 |
|------|------|--------|------|
| `/int` | POST | `{"switch_id":1,"type":1,"end_to_end_latency_ms":150,"jitter_ms":60,"remaining_bandwidth_mbps":"1.2","packet_length":64}` | 模拟 INT 数据上报，触发攻击检测与博弈论响应 |

## 关键注意事项

1. **拟态方法运行在网元上**：前端无法直接访问 s1/s4/s7 的文件系统和日志。需通过以下方式获取数据：
   - **方案 A（推荐）**：在 Controller 上新增代理 API，由 Controller SSH 到各网元读取日志/执行命令后返回 JSON
   - **方案 B**：在各网元上部署轻量 API 服务，前端直接调用网元 API
   - **方案 C**：前端仅展示 Controller `/defense/dashboard` 和 `/mimetic/status` 返回的数据，明细日志通过 WebSocket 或轮询 Controller 代理获取

2. **`GET /defense/dashboard` 是关键聚合 API**，博弈论页面的大部分数据可直接从中获取

3. **数据库密码**：拟态蜜罐的 MySQL 密码为 `Honeypot@2024`，用户 `honeypot`，数据库 `network`，表 `es_honeypot`

4. **IP 地址有两套**：实验网（192.168.x.x）和管理网（10.130.x.x），SSH 通常用实验网 IP

5. **需要 sudo 权限**：3 种拟态方法的启动都需要 sudo（tshark 抓包 / P4 旁路抓包需要 root 权限）
